Test Coverage

test coverage is a percentage measure of the degree to which the source code of a program is executed when a particular test suite is run. A program with high test coverage has more of its source code executed during testing, which suggests it has a lower chance of containing undetected software bugs compared to a program with low test coverage.

Mutation Testing

Mutation testing (or mutation analysis) is used to design new software tests and evaluate the quality of existing software tests. Mutation testing involves modifying a program in small ways.

Mutation Testing

Mutation testing (or mutation analysis) is used to design new software tests and evaluate the quality of existing software tests. Mutation testing involves modifying a program in small ways.

              %%{init: {'theme': 'dark', 'themeVariables': { 'darkMode': true }}}%%
              graph TB
                START((Start)) -->  T[Write tests]
                T --> D[Write code]
                D --> R[Run tests]
                R --> G(All green?)
                G --> |No| D
                G --> |Yes| M[Mutate code]
                subgraph Mutation Testing
                  M -.-> MR[Run tests]
                  MR -.-> MG(All green?)
                  MG -.-> |Yes| MK[Killed]
                  MK -.-> M
                  MG -.-> |No| MS[Survived]
                  MS -.-> M
                end
                M --> END((End))

Goals

  1. identify weakly tested pieces of code
  2. identify weak tests
  3. compute the mutation score: \[\begin{aligned} \frac{killed}{total} \end{aligned}\]

History

  1. 1971, proposed by Richard Lipton as a student
  2. 1978, first developed and published by DeMillo, Lipton and Sayward
  3. 1980, first implementation of a tool by Timothy Budd (PhD work)
